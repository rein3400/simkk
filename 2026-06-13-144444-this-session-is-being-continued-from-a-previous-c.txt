  Root cause

  Setelah deep-fix iterasi, masalahnya adalah try_files fallback /index.php
  di-resolve ke server root (Vue dist), bukan API alias. Jadi request
  /api/health jatuh ke Vue SPA (200 + HTML), sedangkan /api/login (POST) hit
  Laravel tapi dengan script name salah (405).

  Fix (nginx v18)

  Named location @laravel + absolute SCRIPT_FILENAME — tidak bergantung pada
  $realpath_root resolution yang bermasalah:

  location /api {
      alias /var/www/sim-kk/apps/api/public;
      try_files $uri $uri/ @laravel;
      location ~ \.php$ { fastcgi_pass ...; fastcgi_param SCRIPT_FILENAME
  $realpath_root$fastcgi_script_name; }
  }
  location @laravel {
      fastcgi_pass unix:/run/php/php8.3-fpm.sock;
      fastcgi_param SCRIPT_FILENAME /var/www/sim-kk/apps/api/public/index.php;
      include fastcgi_params;
  }

  Plus VITE_API_BASE="" di .env.production → SPA pakai relative /api/*
  (same-origin, no DNS, no CORS).

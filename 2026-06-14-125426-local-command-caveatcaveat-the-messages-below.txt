  🔴 Yang paling bahaya di production (P0)

  1. Admin CRUD routes gak ada role gate — privilege escalation sepele

  apps/api/routes/api.php:81-97

  // Dalam group auth:sanctum — TAPI TANPA role middleware
  Route::get('/admin/layanan', [LayananController::class, 'index']);
  Route::post('/admin/layanan', [LayananController::class, 'store']);
  // ... dst 15 routes tanpa role gate

  Real-world: Kasir yg login bisa POST /admin/users bikin akun Manajer buat
  dirinya sendiri. Bisa hapus semua Layanan. Bisa edit harga Produk. Kasir bisa
  takeover sistem dalam 5 menit.

  Fix: Tambah ->middleware('role:Manajer') di semua 15 route.

  2. Transaction void/reversal 67 baris kode — ZERO test

  apps/api/app/Http/Controllers/Api/TransaksiController.php:86-152

  Void logic: reverse stock FIFO, zero kan buku_kas, update stok_mutasi, audit
  log. Ratusan ribu rupiah di-cancel tanpa verifikasi programmatic bahwa cash
  ledger dan stock balance benar.

  Real-world: Bug di void logic → cash ledger mismatch akhir bulan, stock gak
  balik → stok fisik ≠ stok sistem → audit gagal, manajer gak bisa tutup buku.

  Fix: Tulis 4 test: void Lunas same-day, void prior-day returns 422, void Draft
  hard-delete, hanya Manajer yg boleh void.

  3. SQLite foreign keys mati — data integrity bohongan

  Production runs SQLite, tapi gak ada PRAGMA foreign_keys = ON di config.

  Real-world: transaksi_detail.id_produk bisa nunjuk ke Layanan yg udah dihapus.
  Buku kas bisa punya id_transaksi yg gak exist. Laporan keuangan jadi gak bisa
  di-audit karena referential integrity bohongan.

  Fix: Set 'foreign_key_constraints' => true di config/database.php SQLite
  connection.

  4. Sanctum token gak expired — stolen token = permanent access

  config/sanctum.php:53: 'expiration' => null

  Real-world: Pegawai resign, token masih hidup. Bisa akses API selamanya tanpa
  login lagi. Gak ada cara deteksi.

  Fix: Set 'expiration' => 10080 (7 hari).

  5. Foto klinis upload tanpa size limit — VPS bisa OOM

  RekamMedisController.php:176-188: 'content' => 'required|string' — gak ada
  max.

  Real-world: Upload 100MB base64 foto → PHP allocate ~150MB memory → concurrent
  3 user → OOM → server crash → klinik gak bisa operasi. Plus R2 storage cost
  naik karena foto 50MB padahal 2MB cukup.

  Fix: 'max:10240' (10MB base64 ≈ 7.5MB binary).

  6. SSRF vector via Dompdf isRemoteEnabled + signature_path

  DailyReportService.php:173: $options->set('isRemoteEnabled', true) — Dompdf
  fetch external resources.

  Kombinasi: user bisa set signature_path=http://169.254.169.254/... (meta-data
  server) atau signature_path=file:///etc/passwd → muncul di PDF.

  Fix: isRemoteEnabled = false, render signature sebagai inline base64.

  7. Double-spend race condition — idempotency key tanpa UNIQUE constraint

  TransaksiController.php:41-47 — dua request concurrent bisa sama-sama lewat
  first() check → double charge pasien, double decrement stock.

  Real-world: Kasir klik Lunas, loading lemot, klik lagi → pasien di-charge 2x,
  stok berkurang 2x. Komplain pasien, refund ribet.

  Fix: Unique index di (user_id, key_hash, endpoint) + catch duplicate key
  exception.

  8. transaksi_detail.id_produk FK nunjuk ke layanan — naming sesat

  migration:14: $table->foreignId('id_produk')->constrained('layanan')

  Real-world: Developer baru baca kode, pikir id_produk = produk fisik,
  sebenernya Layanan. Bug reporting salah kategorisasi.

  🟠 Real-world problems (P1)

  9. No rate limit di login — brute force sepele

  routes/api.php:21: Route::post('/login', ...) tanpa throttle. Password semua
  user simkk-2026.

  Real-world: Script kiddie brute force login dalam hitungan jam. Begitu masuk →
  privilege escalation (P0#1).

  10. Password minimum 6 karakter — NIST standard gagal

  UserAdminController.php:134: 'min:6'

  Klinik staf pake 123456 atau nama sendiri. Bruteforce cepat.

  11. Treatment date disimpan sebagai "14 Jun" — tanpa tahun

  RekamMedisController.php:56: now()->format('d M')

  Real-world: Juni 2027 treatment "14 Jun" — gak bisa dibedakan sama Juni 2026.
  Riwayat medis tidak bisa diurutkan kronologis dengan benar. Malpraktik risk
  karena salah baca timeline treatment.

  12. BootstrapController dump full database ke semua role

  BootstrapController.php:21: /api/bootstrap balikin semua pasien, semua
  transaksi, semua treatment notes, signed URL foto — ke Terapis, Kasir, Gudang,
  siapa pun.

  Real-world: Kasir bisa lihat catatan treatment dan foto klinis pasien (privacy
  violation). Terapis bisa lihat total transaksi semua pasien. Data leakage
  masif.

  Fix: Scope response by role atau restrict ke role:Manajer.

  13. Telegram phone match 9-digit terakhir — salah pasien

  TelegramController.php:183: LIKE '%' + substr(digits, -9) — kalau 2 pasien
  share 9 digit terakhir (keluarga, SIM card gantian), chat_id nempel ke pasien
  salah.

  Real-world: Notif aftercare + reminder kirim ke orang salah → nama pasien +
  detail treatment bocor ke orang lain. HIPAA/privacy nightmare.

  14. updateTreatment / deleteTreatment — code jalan tapi gak ada route

  RekamMedisController.php:76,118: implementasi lengkap, authorization checks
  ada — tapi gak ada route yg map ke method ini.

  Real-world: Terapis gak bisa edit catatan yg salah ketik. Gak bisa hapus
  treatment yg salah input. Harus panggil support/manajer untuk hapus manual di
  DB.

  15. tanggal di last_visit, catatan_treatment.tanggal, foto_klinis.tanggal 
  pakai string, bukan date column

  Multiple migrations: string(20) padahal isinya tanggal.

  Real-world: ORDER BY tanggal → Apr, Aug, Dec, Feb, Jan... urutan salah. Query
  WHERE tanggal > '2026-01-01' return garbage. Laporan rekam medis rusak.

  16. Soft deletes missing di semua critical table

  Pasien, Terapis, Transaksi, Detail, BukuKas — gak ada deleted_at.

  Real-world: Operator salah klik delete pasien → riwayat transaksi + treatment
  lenyap permanen. Laporan keuangan gak bisa diaudit. Gak comply sama regulasi
  penyimpanan data medis (minimal retention period).

  17. Internal RuntimeException messages bocor ke API consumer

  bootstrap/app.php:28-38: raw exception message balik ke JSON 409.

  Real-world: Error SQLite path, file structure, table names — terekspos ke
  attacker. Reconnaissance gratis.

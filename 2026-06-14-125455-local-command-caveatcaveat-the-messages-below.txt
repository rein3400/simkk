  2.4 Security Vulnerabilities

  CRITICAL:

  1. Auth Lifecycle Incomplete
    - No session expiry — tokens valid forever if stolen
    - No password reset — users locked out permanently
    - No 2FA for admin/manajer roles
    - Token rotation not implemented
  2. Medical Photo Exposure
    - Photos stored locally, no access control
    - No audit trail for photo access
    - No consent mechanism documented
    - Violation: UU Nomor 27 Tahun 2022 tentang Perlindungan Data Pribadi
  3. Audit Logging Gaps
    - Only login events audited
    - Transaction modifications not audited
    - Photo uploads/deletions not audited
    - User/role changes not audited
  4. Input Validation
    - Upload validation not hardened (size/format limits untested)
    - Commission snapshot immutability not enforced at DB level
    - No rate limiting documented
  5. Secret Management
    - .env file untracked (good) but no secret rotation policy
    - S3/R2 credentials stored in plain text
    - No Vault/KMS reference

  2.6 Business Logic Verification

  Aturan bisnis yang tidak terjamin:

  1. FIFO Stock Valuation
    - Sort key: ORDER BY CASE WHEN expiry = 'Reusable' THEN '9999-12-31' ELSE 
  expiry END ASC, id ASC
    - Only partially tested — edge cases across batch boundaries unverified
    - HPP calculation critical for commission & profit reporting
  2. Commission Immutability
    - Rule: "Nilai_komisi snapshot at Lunas, never recompute"
    - Not enforced at database level (no trigger/constraint)
    - Test coverage incomplete — only happy path tested
  3. Report Auditability
    - Claim: "Reports must derive from approved history, not manual entry"
    - Verification: PDF/Excel export tests exist but don't verify source data
  integrity
    - Risk: If someone manually edits report values post-generation, no
  detection
  4. Inventory Batch & Expiry
    - No detailed schema for batches, expiry dates, supplier purchases
    - FIFO requires proper mutation ledger — incomplete
    - Stock adjustment/write-off paths untested
